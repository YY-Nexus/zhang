# UI提示词概览

---

## 任务卡（按优先级排序）

### 任务卡 01 首页 Hero 与 3D 无框画廊
- **角色**：UI 设计师、3D 设计师、前端工程师  
- **估时**：设计 3 天；前端实现 6 天（合计 9 天）  
- **优先级**：高  
- **任务描述**：设计并实现首页 Hero 区，包含 Three.js 无框画廊墙面、倒计时、左上 Logo 映射浮动 AI 按钮入口；画廊支持点击打开弹窗查看大图并触发对应音乐。移动端自动降级为平面滑动视图。  
- **验收步骤**：  
  1. Figma 输出 Hero 高保真稿与交互说明（含光照、反射、漫游路径）。  
  2. Three.js 原型在桌面浏览器运行，首屏渲染时间 < 2s（模拟 3G/4G 降级策略）。  
  3. 点击画作能打开弹窗并播放对应曲目（或触发播放 API）。  
  4. 移动端显示平面滑动视图且触控滑动顺畅。

#### 任务卡 02 微透可拖拽弹窗系统
- **角色**：前端工程师、动效工程师、无障碍工程师  
- **估时**：设计 2 天；实现 5 天（合计 7 天）  
- **优先级**：高  
- **任务描述**：实现通用弹窗组件（微透、可拖拽、边界约束、最小化/最大化、背景点击或 Esc 关闭、键盘无障碍）。提供变体：音乐播放器、RSVP、留言墙、来宾名单、地图导航、AI 面板。  
- **验收步骤**：  
  1. 提供 TSX 组件骨架与 props 文档（title,width,height,draggable,onClose,ariaLabel）。  
  2. 拖拽手柄可拖动并受边界约束；最小化/最大化功能可用。  
  3. 点击背景或按 Esc 能关闭弹窗；键盘导航与屏幕阅读器标签通过基本无障碍检查。  
  4. 弹窗动画（缩放+模糊+阴影）在高/低性能设备均有降级策略。

#### 任务卡 03 左上浮动 AI 智能交互按钮與面板
- **角色**：产品、AI 文案、前端工程师  
- **估时**：文案与对话流 1.5 天；UI/前端实现 3 天（合计 4.5 天）  
- **优先级**：高  
- **任务描述**：实现 Logo 底层映射的常驻微透 AI 按钮，点击弹出可拖拽 AI 面板，支持语音输入、快捷命令（导航、签到、播放音乐、RSVP、一键呼叫）、表情建议与状态持久化。对话语气热情和谐，兼顾洛阳习俗提示。  
- **验收步骤**：  
  1. 提供 6 条开场话术与 6 条快捷命令 JSON（含命令标识）。  
  2. AI 面板可拖拽、最小化并在页面间保持状态。  
  3. 语音输入能转文本并触发对应快捷命令（模拟触发测试）。  
  4. 面板在移动端与桌面端均可用且无遮挡关键内容。

#### 任务卡 04 中文音乐播放器与投票系统
- **角色**：前端工程师、后端工程师、音乐策划  
- **估时**：曲库整理 1.5 天；前端实现 3 天；后端接口 2 天（合计 6.5 天）  
- **优先级**：高  
- **任务描述**：实现分段播放（进场/敬茶/用餐/送客）、淡入淡出、来宾投票、跨设备播放位置同步与音量记忆。管理员可编辑播放列表与优先级。  
- **验收步骤**：  
  1. 提供 20 首中文曲目清单并按环节标注情绪标签。  
  2. 前端实现 voteTrack(trackId, guestId) 与 syncPosition(sessionId) 接口调用示例。  
  3. 投票结果能实时影响播放优先级（模拟并发投票测试）。  
  4. 播放器在切换环节时平滑淡入淡出且无明显卡顿。

#### 任务卡 05 日程与地点页面 一键导航与呼叫
- **角色**：前端工程师、后端工程师、产品  
- **估时**：设计与集成 2 天；测试 1 天（合计 3 天）  
- **优先级**：中高  
- **任务描述**：实现日程展示（农历/公历并列）、嵌入地图卡片（显示富豪大酒店地址与坐标）、一键导航（唤起本地地图）与一键呼叫（拨打新郎/新娘/联系人并弹出确认）。记录呼叫日志。  
- **验收步骤**：  
  1. 地点卡显示地址与坐标并能唤起 Apple/高德/百度地图（移动端测试）。  
  2. 一键呼叫弹出确认并能触发拨号；呼叫日志写入后端。  
  3. 农历与公历显示正确且格式清晰。

#### 任务卡 06 RSVP 表单與来宾管理 UI
- **角色**：产品、前端工程师、后端工程师  
- **估时**：表单设计 1 天；前端实现 2 天；后端 API 2 天（合计 5 天）  
- **优先级**：高  
- **任务描述**：实现智能 RSVP 表单（姓名、电话、出席状态、餐饮偏好、座位需求、备注），支持来宾导入/导出、批量编辑与 RSVP 报表。表单支持保存草稿与提醒。  
- **验收步骤**：  
  1. 表单字段完整并有前端校验（电话格式、必填项）。  
  2. POST /rsvp 接口能接收并返回成功状态；后台能导出 CSV。  
  3. 管理端能查看 RSVP 报表并按餐饮偏好分组。

#### 任务卡 07 留言墙與实时聊天室
- **角色**：前端工程师、后端工程师、QA  
- **估时**：前端 2 天；后端实时服务 3 天；测试 1 天（合计 6 天）  
- **优先级**：中  
- **任务描述**：实现留言墙（持久化）與实时聊天室（WebSocket），支持消息广播、在线来宾列表、点赞、表情与投票功能。提供消息持久化与简单防刷机制。  
- **验收步骤**：  
  1. 实时消息延迟 < 1s（本地网络条件下）。  
  2. 在线列表能实时更新并显示来宾昵称。  
  3. 留言墙消息持久化并能分页加载；点赞功能生效。

#### 任务卡 08 交互动效系统与动效库
- **角色**：动效工程师、前端工程师  
- **估时**：规范文档 2 天；实现常用动效组件 3 天（合计 5 天）  
- **优先级**：中高  
- **任务描述**：制定全站动效规范（流体过渡、光晕、微振动、按钮反馈），使用 GSAP/FLIP 实现关键动画并提供降级开关。输出动效组件库（Modal 动画、按钮、画廊切换、AI 弹出）。  
- **验收步骤**：  
  1. 动效规范文档包含 easing、时长、触发条件与降级策略。  
  2. 关键动效在桌面 60fps 下流畅；低性能设备可关闭复杂动画。  
  3. 提供 5 个可复用动效组件示例。

#### 任务卡 09 设计系统與 UI Tokens
- **角色**：设计师、前端工程师  
- **估时**：设计 tokens 1.5 天；组件库初版 3 天（合计 4.5 天）  
- **优先级**：中  
- **任务描述**：建立颜色、间距、字体、阴影、圆角等 Design Tokens；输出 Figma 组件库与前端可用样式变量（CSS/SCSS 或 Tailwind 配置）。  
- **验收步骤**：  
  1. 提供 tokens JSON 与 Figma 组件库链接（或文件）。  
  2. 前端能通过 tokens 快速复用样式并与设计稿一致。  
  3. 包含暗色/高对比模式切换方案。

#### 任务卡 10 响应式与无障碍适配
- **角色**：前端工程师、无障碍工程师、QA  
- **估时**：实现与测试 4 天  
- **优先级**：高  
- **任务描述**：移动优先设计，触控手势友好，键盘导航、ARIA 标签、屏幕阅读器支持与高对比模式。确保弹窗、画廊、播放器在触控设备上可用。  
- **验收步骤**：  
  1. 通过基本无障碍检查（键盘可达、aria-label 存在）。  
  2. 移动端触控拖拽与长按提示正常。  
  3. 高对比模式下文本可读性满足 WCAG 对比度建议。

#### 任务卡 11 页面性能优化与资源管理
- **角色**：前端工程师、运维  
- **估时**：优化策略 2 天；实施 3 天（合计 5 天）  
- **优先级**：中高  
- **任务描述**：实现图片/模型懒加载、Three.js LOD、动画合批、CDN 与缓存策略，目标首屏渲染 < 2s。  
- **验收步骤**：  
  1. Lighthouse 首屏性能评分达到目标（可设具体阈值）。  
  2. 模型与图片按需加载并有占位图。  
  3. CDN 配置并验证缓存命中率。

#### 任务卡 12 页面测试与验收清单
- **角色**：QA、产品、前端  
- **估时**：编写测试用例 1.5 天；执行回归 2 天（合计 3.5 天）  
- **优先级**：高（上线前必须）  
- **任务描述**：编写并执行覆盖弹窗、拖拽、音乐、RSVP、聊天、导航、呼叫、无障碍与性能的测试用例。输出测试报告与缺陷清单。  
- **验收步骤**：  
  1. 提供 >= 50 条测试用例并执行记录。  
  2. 所有阻断级缺陷修复并复测通过。  
  3. 性能与无障碍关键项满足验收标准。

### 交付节奏与并行建议

- **并行化**：可同时推进任务卡 01、02、03（设计与前端原型并行），任务卡 04、06、07 可并行开发后端与前端接口。  
- **总估时（串行）**：若按顺序完成，约 58 天；实际并行可压缩至 4–6 周交付 MVP（核心功能：Hero、弹窗、AI 面板、音乐、RSVP）。  
- **里程碑建议**：  
  - 第 1 周：视觉稿与 3D 原型（任务卡 01、02 设计）  
  - 第 2–3 周：前端核心实现（画廊、弹窗、AI 面板、播放器）  
  - 第 4 周：后端接口、实时服务与集成测试  
  - 第 5 周：性能优化、无障碍与上线准备

- 后端 API 文档（OpenAPI 简要）
